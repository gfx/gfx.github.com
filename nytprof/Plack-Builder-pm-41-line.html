117=head1 NAME
118
119Plack::Builder - OO and DSL to enable Plack Middlewares
120
121=head1 SYNOPSIS
122
123 # in .psgi
124 use Plack::Builder;
125
126 my $app = sub { ... };
127
128 builder {
129 enable "Plack::Middleware::Foo";
130 enable "Plack::Middleware::Bar", opt => "val";
131 enable "Plack::Middleware::Baz";
132 $app;
133 };
134
135 # use URLMap
136
137 builder {
138 mount "/foo" => builder {
139 enable "Plack::Middleware::Foo";
140 $app;
141 };
142
143 mount "/bar" => $app2;
144 mount "http://example.com/" => builder { $app3 };
145 };
146
147 # using OO interface
148
149 my $builder = Plack::Builder->new();
150 $builder->add_middleware('Foo', opt => 1);
151 $app = $builder->mount('/app' => $app);
152 $app = $builder->to_app($app);
153
154=head1 DESCRIPTION
155
156Plack::Builder gives you a quick domain specific language (DSL) to
157wrap your application with Plack::Middleware subclasses. The
158middleware you're trying to use should use L<Plack::Middleware> as a
159base class to use this DSL, inspired by Rack::Builder.
160
161Whenever you call C<enable> on any middleware, the middleware app is
162pushed to the stack inside the builder, and then reversed when it
163actually creates a wrapped application handler, so:
164
165 builder {
166 enable "Plack::Middleware::Foo";
167 enable "Plack::Middleware::Bar", opt => "val";
168 $app;
169 };
170
171is syntactically equal to:
172
173 $app = Plack::Middleware::Bar->wrap($app, opt => "val");
174 $app = Plack::Middleware::Foo->wrap($app);
175
176In other words, you're supposed to C<enable> middleware from outer to inner.
177
178=head1 INLINE MIDDLEWARE
179
180Plack::Builder allows you to code middleware inline using a nested
181code reference.
182
183If the first argument to C<enable> is a code reference, it will be
184passed an C<$app> and is supposed to return another code reference
185which is PSGI application that consumes C<$env> in runtime. So:
186
187 builder {
188 enable sub {
189 my $app = shift;
190 sub {
191 my $env = shift;
192 # do preprocessing
193 my $res = $app->($env);
194 # do postprocessing
195 return $res;
196 };
197 };
198 $app;
199 };
200
201is equal to:
202
203 my $mw = sub {
204 my $app = shift;
205 sub { my $env = shift; $app->($env) };
206 };
207
208 $app = $mw->($app);
209
210=head1 URLMap support
211
212Plack::Builder has a native support for L<Plack::App::URLMap> with C<mount> method.
213
214 use Plack::Builder;
215 my $app = builder {
216 mount "/foo" => $app1;
217 mount "/bar" => builder {
218 enable "Plack::Middleware::Foo";
219 $app2;
220 };
221 };
222
223See L<Plack::App::URLMap>'s C<map> method to see what they mean. With
224builder you can't use C<map> as a DSL, for the obvious reason :)
225
226B<NOTE>: Once you use C<mount> in your builder code, you have to use
227C<mount> for all the paths, including the root path (C</>). You can't
228have the default app in the last line of C<builder> like:
229
230 my $app = sub {
231 my $env = shift;
232 ...
233 };
234
235 builder {
236 mount "/foo" => sub { ... };
237 $app; # THIS DOESN'T WORK
238 };
239
240You'll get warnings saying that your mount configuration will be
241ignored. Instead you should use C<< mount "/" => ... >> in the last
242line to set the default fallback app.
243
244 builder {
245 mount "/foo" => sub { ... };
246 mount "/" => $app;
247 }
248
249Note that the C<builder> DSL returns a whole new PSGI application, which means
250
251=over 4
252
253=item *
254
255C<builder { ... }> should normally the last statement of a C<.psgi>
256file, because the return value of C<builder> is the application that
257actually is executed.
258
259=item *
260
261You can nest your C<builder> block, mixed with C<mount> (see URLMap
262support above):
263
264 builder {
265 mount "/foo" => builder {
266 mount "/bar" => $app;
267 }
268 }
269
270will locate the C<$app> under C</foo/bar> since the inner C<builder>
271block puts it under C</bar> and it results a new PSGI application
272which is located under C</foo> because of the outer C<builder> block.
273
274=back
275
276=head1 CONDITIONAL MIDDLEWARE SUPPORT
277
278You can use C<enable_if> to conditionally enable middleware based on
279the runtime environment. See L<Plack::Middleware::Conditional> for
280details.
