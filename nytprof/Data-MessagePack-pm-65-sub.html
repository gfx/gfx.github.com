61=head1 NAME
62
63Data::MessagePack - MessagePack serialising/deserialising
64
65=head1 SYNOPSIS
66
67 use Data::MessagePack;
68
69 my $mp = Data::MessagePack->new();
70 $mp->canonical->utf8->prefer_integer if $needed;
71
72 my $packed = $mp->pack($dat);
73 my $unpacked = $mp->unpack($dat);
74
75=head1 DESCRIPTION
76
77This module converts Perl data structures to MessagePack and vice versa.
78
79=head1 ABOUT MESSAGEPACK FORMAT
80
81MessagePack is a binary-based efficient object serialization format.
82It enables to exchange structured objects between many languages like JSON.
83But unlike JSON, it is very fast and small.
84
85=head2 ADVANTAGES
86
87=over 4
88
89=item PORTABLE
90
91The MessagePack format does not depend on language nor byte order.
92
93=item SMALL IN SIZE
94
95 say length(JSON::XS::encode_json({a=>1, b=>2})); # => 13
96 say length(Storable::nfreeze({a=>1, b=>2})); # => 21
97 say length(Data::MessagePack->pack({a=>1, b=>2})); # => 7
98
99The MessagePack format saves memory than JSON and Storable format.
100
101=item STREAMING DESERIALIZER
102
103MessagePack supports streaming deserializer. It is useful for networking such as RPC.
104See L<Data::MessagePack::Unpacker> for details.
105
106=back
107
108If you want to get more information about the MessagePack format, please visit to L<http://msgpack.org/>.
109
110=head1 METHODS
111
112=over
113
114=item C<< my $packed = Data::MessagePack->pack($data[, $max_depth]); >>
115
116Pack the $data to messagepack format string.
117
118This method throws an exception when the perl structure is nested more than $max_depth levels(default: 512) in order to detect circular references.
119
120Data::MessagePack->pack() throws an exception when encountering blessed object, because MessagePack is language-independent format.
121
122=item C<< my $unpacked = Data::MessagePack->unpack($msgpackstr); >>
123
124unpack the $msgpackstr to a MessagePack format string.
125
126=item C<< my $mp = Data::MesssagePack->new() >>
127
128Creates a new MessagePack instance.
129
130=item C<< $mp = $mp->prefer_integer([ $enable ]) >>
131
132=item C<< $enabled = $mp->get_prefer_integer() >>
133
134If I<$enable> is true (or missing), then the C<pack> method tries a string
135as an integer if the string looks like an integer.
136
137=item C<< $mp = $mp->canonical([ $enable ]) >>
138
139=item C<< $enabled = $mp->get_canonical() >>
140
141If I<$enable> is true (or missing), then the C<pack> method will output
142packed data by sorting their keys. This is adding a comparatively high
143overhead.
144
145=item C<< $mp = $mp->utf8([ $enable ]) >>
146
147=item C<< $enabled = $mp->get_utf8() >>
148
149If I<$enable> is true (or missing), then the C<pack> method will
150apply C<utf8::encode()> to all the string values.
151
152In other words, this property tell C<$mp> to deal with B<text strings>.
153See L<perlunifaq> for the meaning of B<text string>.
154
155=item C<< $packed = $mp->pack($data) >>
156
157=item C<< $packed = $mp->encode($data) >>
158
159Same as C<< Data::MessagePack->pack() >>, but properties are respected.
160
161=item C<< $data = $mp->unpack($data) >>
162
163=item C<< $data = $mp->decode($data) >>
164
165Same as C<< Data::MessagePack->unpack() >>, but properties are respected.
166
167=back
168

213=head1 CAVEAT
214
215=head2 Unpacking 64 bit integers
216
217This module can unpack 64 bit integers even if your perl does not support them
218(i.e. where C<< perl -V:ivsize >> is 4), but you cannot calculate these values
219unless you use C<Math::BigInt>.
220
221=head1 TODO
222
223=over
224
225=item Error handling
226
227MessagePack cannot deal with complex scalars such as object references,
228filehandles, and code references. We should report the errors more kindly.
229
230=item Streaming deserializer
231
232The current implementation of the streaming deserializer does not have internal
233buffers while some other bindings (such as Ruby binding) does. This limitation
234will astonish those who try to unpack byte streams with an arbitrary buffer size
235(e.g. C<< while(read($socket, $buffer, $arbitrary_buffer_size)) { ... } >>).
236We should implement the internal buffer for the unpacker.
237
238=back
