579=head1 NAME
580
581Text::Xslate - Scalable template engine for Perl5
582
583=head1 VERSION
584
585This document describes Text::Xslate version 1.4001.
586
587=head1 SYNOPSIS
588
589 use Text::Xslate qw(mark_raw);
590
591 my $tx = Text::Xslate->new();
592
593 my %vars = (
594 title => 'A list of books',
595 books => [
596 { title => 'Islands in the stream' },
597 { title => 'Programming Perl' },
598 # ...
599 ],
600
601 # mark HTML components as raw not to escape its HTML tags
602 gadget => mark_raw('<div class="gadget">...</div>'),
603 );
604
605 # for files
606 print $tx->render('hello.tx', \%vars);
607
608 # for strings
609 my $template = q{
610 <h1><: $title :></h1>
611 <ul>
612 : for $books -> $book {
613 <li><: $book.title :></li>
614 : } # for
615 </ul>
616 };
617
618 print $tx->render_string($template, \%vars);
619
620=head1 DESCRIPTION
621
622B<Xslate> is a template engine, tuned for persistent applications,
623safe as an HTML generator, and with rich features.
624
625There are a lot of template engines in CPAN, for example Template-Toolkit,
626Text::MicroTemplate, HTML::Template, and so on, but all of them have
627demerits at some points. This is why Xslate is developed and now it is
628well-honed as the standard template engine for web applications.
629
630The concept of Xslate is strongly influenced by Text::MicroTemplate
631and Template-Toolkit 2, but the central philosophy of Xslate is different
632from them. That is, the philosophy is B<sandboxing> that the template logic
633should not have no access outside the template beyond your permission.
634
635Other remarkable features are as follows:
636
637=head2 Features
638
639=head3 High performance
640
641This engine introduces the virtual machine paradigm. Templates are
642compiled into intermediate code, and then executed by the virtual machine,
643which is highly optimized for rendering templates. Thus, Xslate is
644much faster than any other template engines.
645
646The template roundup project by Sam Graham shows Text::Xslate got very
647high scores in I<instance_reuse> condition (i.e. for persistent application).
648
649=over
650
651=item The template roundup project
652
653L<http://illusori.co.uk/projects/Template-Roundup/>
654
655=item Perl Template Roundup October 2010 Performance vs Variant Report: instance_reuse
656
657L<http://illusori.co.uk/projects/Template-Roundup/201010/performance_vs_variant_by_feature_for_instance_reuse.html>
658
659=back
660
661There are some benchmarks in F<benchmark/> directory in the Xslate distribution.
662
663=head3 Smart escaping for HTML meta characters
664
665All the HTML meta characters in template expressions the engine interpolates
666into template texts are escaped automatically, so the output has no
667possibility to XSS by default.
668
669=head3 Template cascading
670
671Xslate supports B<template cascading>, which allows you to extend
672templates with block modifiers. It is like traditional template inclusion,
673but is more powerful.
674
675This mechanism is also called as template inheritance.
676
677=head3 Easiness to enhance
678
679Xslate is ready to enhance. You can add functions and methods to the template
680engine and even add a new syntax via extending the parser.
681
682=head1 INTERFACE
683
684=head2 Methods
685
686=head3 B<< Text::Xslate->new(%options) :XslateEngine >>
687
688Creates a new xslate template engine with options. You can reuse the instance
689for multiple call of C<render()>.
690
691Possible options are:
692
693=over
694
695=item C<< path => \@path // ['.'] >>
696
697Specifies the include paths, which may be directory names or virtual paths,
698i.e. HASH references which contain C<< $file_name => $content >> pairs.
699
700=item C<< cache => $level // 1 >>
701
702Sets the cache level.
703
704If I<$level> == 1 (default), Xslate caches compiled templates on the disk, and
705checks the freshness of the original templates every time.
706
707If I<$level> E<gt>= 2, caches will be created but the freshness
708will not be checked.
709
710I<$level> == 0 uses no caches, which is provided for testing.
711
712=item C<< cache_dir => $dir // "$ENV{HOME}/.xslate_cache" >>
713
714Specifies the directory used for caches. If C<$ENV{HOME}> doesn't exist,
715C<< File::Spec->tmpdir >> will be used.
716
717You B<should> specify this option for productions to avoid conflicts of
718template names.
719
720=item C<< function => \%functions >>
721
722Specifies a function map which contains name-coderef pairs.
723A function C<f> may be called as C<f($arg)> or C<$arg | f> in templates.
724
725There are built-in functions described in L<Text::Xslate::Manual::Builtin>.
726
727=item C<< module => [$module => ?\@import_args, ...] >>
728
729Imports functions from I<$module>, which may be a function-based or bridge module.
730Optional I<@import_args> are passed to C<import> as C<< $module->import(@import_args) >>.
731
732For example:
733
734 # for function-based modules
735 my $tx = Text::Xslate->new(
736 module => ['Time::Piece'],
737 );
738 print $tx->render_string(
739 '<: localtime($x).strftime() :>',
740 { x => time() },
741 ); # => Wed, 09 Jun 2010 10:22:06 JST
742
743 # for bridge modules
744 my $tx = Text::Xslate->new(
745 module => ['SomeModule::Bridge::Xslate'],
746 );
747 print $tx->render_string(
748 '<: $x.some_method() :>',
749 { x => time() },
750 );
751
752Because you can use function-based modules with the C<module> option, and
753also can invoke any object methods in templates, Xslate doesn't require
754specific namespaces for plugins.
755
756=item C<< input_layer => $perliolayers // ':utf8' >>
757
758Specifies PerlIO layers to open template files.
759
760=item C<< verbose => $level // 1 >>
761
762Specifies the verbose level.
763
764If C<< $level == 0 >>, all the possible errors will be ignored.
765
766If C<< $level> >= 1 >> (default), trivial errors (e.g. to print nil) will be ignored,
767but severe errors (e.g. for a method to throw the error) will be warned.
768
769If C<< $level >= 2 >>, all the possible errors will be warned.
770
771=item C<< suffix => $ext // '.tx' >>
772
773Specify the template suffix, which is used for C<cascade> and C<include>
774in Kolon.
775
776Note that this is used for static name resolution. That is, the compiler
777uses it but the runtime engine doesn't.
778
779=item C<< syntax => $name // 'Kolon' >>
780
781Specifies the template syntax you want to use.
782
783I<$name> may be a short name (e.g. C<Kolon>), or a fully qualified name
784(e.g. C<Text::Xslate::Syntax::Kolon>).
785
786This option is passed to the compiler directly.
787
788=item C<< type => $type // 'html' >>
789
790Specifies the output content type. If I<$type> is C<html> or C<xml>,
791smart escaping is applied to template expressions. That is,
792they are interpolated via the C<html_escape> filter.
793If I<$type> is C<text> smart escaping is not applied so that it is
794suitable for plain texts like e-mails.
795
796I<$type> may be B<html>, B<xml> (identical to C<html>), and B<text>.
797
798This option is passed to the compiler directly.
799
800=item C<< line_start => $token // $parser_defined_str >>
801
802Specify the token to start line code as a string, which C<quotemeta> will be applied to. If you give C<undef>, the line code style is disabled.
803
804This option is passed to the parser via the compiler.
805
806=item C<< tag_start => $str // $parser_defined_str >>
807
808Specify the token to start inline code as a string, which C<quotemeta> will be applied to.
809
810This option is passed to the parser via the compiler.
811
812=item C<< tag_end => $str // $parser_defined_str >>
813
814Specify the token to end inline code as a string, which C<quotemeta> will be applied to.
815
816This option is passed to the parser via the compiler.
817
818=item C<< header => \@template_files >>
819
820Specify the header template files, which are inserted to the head of each template.
821
822This option is passed to the compiler.
823
824=item C<< footer => \@template_files >>
825
826Specify the footer template files, which are inserted to the foot of each template.
827
828This option is passed to the compiler.
829
830=item C<< warn_handler => \&cb >>
831
832Specify the callback I<&cb> which is called on warnings.
833
834This option is experimental.
835
836=item C<< die_handler => \&cb >>
837
838Specify the callback I<&cb> which is called on fatal errors.
839
840This option is experimental.
841
842=back

846Renders a template file with given variables, and returns the result.
847I<\%vars> is optional.

849Note that I<$file> may be cached according to the cache level.

851=head3 B<< $tx->render_string($string, \%vars) :Str >>

887This method is significant when it is called by template functions and methods.

893=head2 Exportable functions
894
895=head3 C<< mark_raw($str :Str) :RawStr >>
896
897Marks I<$str> as raw, so that the content of I<$str> will be rendered as is,
898so you have to escape these strings by yourself.
899
900For example:
901
902 my $tx = Text::Xslate->new();
903 my $tmpl = 'Mailaddress: <: $email :>';
904 my %vars = (
905 email => mark_raw('Foo &lt;foo at example.com&gt;'),
906 );
907 print $tx->render_string($tmpl, \%email);
908 # => Mailaddress: Foo &lt;foo@example.com&gt;
909
910This function is available in templates as the C<mark_raw> filter, although
911the use of it is strongly discouraged.
912
913=head3 C<< unmark_raw($str :Str) :Str >>
914
915Clears the raw marker from I<$str>, so that the content of I<$str> will
916be escaped before rendered.
917
918This function is available in templates as the C<unmark_raw> filter.
919
920=head3 C<< html_escape($str :Str) :RawStr >>
921
922Escapes HTML meta characters in I<$str>, and returns it as a raw string (see above).
923If I<$str> is already a raw string, it returns I<$str> as is.
924
925By default, this function will be automatically applied to all the template
926expressions.
927
928This function is available in templates as the C<html> filter, but you'd better
929to use C<unmark_raw> to ensure expressions to be html-escaped.
930
931=head3 C<< uri_escape($str :Str) :Str >>
932
933Escapes URI unsafe characters in I<$str>, and returns it.
934
935This function is available in templates as the C<uri> filter.
936
937=head3 C<< html_builder { block } | \&function :CodeRef >>
938
939Wraps a block or I<&function> with C<mark_raw> so that the new subroutine
940will return a raw string.
941
942This function is used to tell the xslate engine that I<&function> is an
943HTML builder that returns HTML sources. For example:
944
945 sub some_html_builder {
946 my @args = @_;
947 my $html;
948 # build HTML ...
949 return $html;
950 }
951
952 my $tx = Text::Xslate->new(
953 function => {
954 some_html_builder => html_builder(\&some_html_builder),
955 },
956 );
957
958See also L<Text::Xslate::Manual::Cookbook>.

960=head2 Command line interface
961

973=head1 TEMPLATE SYNTAX
974
975There are multiple template syntaxes available in Xslate.
976
977=over
978
979=item Kolon
980
981B<Kolon> is the default syntax, using C<< <: ... :> >> inline code and
982C<< : ... >> line code, which is explained in L<Text::Xslate::Syntax::Kolon>.
983
984=item Metakolon
985
986B<Metakolon> is the same as Kolon except for using C<< [% ... %] >> inline code and
987C<< %% ... >> line code, instead of C<< <: ... :> >> and C<< : ... >>.
988
989=item TTerse
990
991B<TTerse> is a syntax that is a subset of Template-Toolkit 2 (and partially TT3),
992which is explained in L<Text::Xslate::Syntax::TTerse>.
993
994=item HTMLTemplate
995
996There's HTML::Template compatible layers in CPAN.
997
998L<Text::Xslate::Syntax::HTMLTemplate> is a syntax for HTML::Template.
999
1000L<HTML::Template::Parser> is a converter from HTML::Template to Text::Xslate.
1001
1002=back

1008=head2 Nil/undef handling
1009
1010Note that nil (i.e. C<undef> in Perl) handling is different from Perl's.
1011Basically it does nothing, but C<< verbose => 2 >> will produce warnings on it.

1017Prints nothing.
1018
1019=item to access fields
1020
1021Returns nil. That is, C<< nil.foo.bar.baz >> produces nil.
1022
1023=item to invoke methods
1024
1025Returns nil. That is, C<< nil.foo().bar().baz() >> produces nil.
1026
1027=item to iterate
1028
1029Dealt as an empty array.
1030
1031=item equality
1032
1033C<< $var == nil >> returns true if and only if I<$var> is nil.
