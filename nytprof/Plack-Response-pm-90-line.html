169=head1 NAME
170
171Plack::Response - Portable HTTP Response object for PSGI response
172
173=head1 SYNOPSIS
174
175 use Plack::Response;
176
177 sub psgi_handler {
178 my $env = shift;
179
180 my $res = Plack::Response->new(200);
181 $res->content_type('text/html');
182 $res->body("Hello World");
183
184 return $res->finalize;
185 }
186
187=head1 DESCRIPTION
188
189Plack::Response allows you a way to create PSGI response array ref through a simple API.
190
191=head1 METHODS
192
193=over 4
194
195=item new
196
197 $res = Plack::Response->new;
198 $res = Plack::Response->new($status);
199 $res = Plack::Response->new($status, $headers);
200 $res = Plack::Response->new($status, $headers, $body);
201
202Creates a new Plack::Response object.
203
204=item status
205
206 $res->status(200);
207 $status = $res->status;
208
209Sets and gets HTTP status code. C<code> is an alias.
210
211=item headers
212
213 $headers = $res->headers;
214 $res->headers([ 'Content-Type' => 'text/html' ]);
215 $res->headers({ 'Content-Type' => 'text/html' });
216 $res->headers( HTTP::Headers->new );
217
218Sets and gets HTTP headers of the response. Setter can take either an
219array ref, a hash ref or L<HTTP::Headers> object containing a list of
220headers.
221
222=item body
223
224 $res->body($body_str);
225 $res->body([ "Hello", "World" ]);
226 $res->body($io);
227
228Gets and sets HTTP response body. Setter can take either a string, an
229array ref, or an IO::Handle-like object. C<content> is an alias.
230
231Note that this method doesn't automatically set I<Content-Length> for
232the response. You have to set it manually if you want, with the
233C<content_length> method (see below).
234
235=item header
236
237 $res->header('X-Foo' => 'bar');
238 my $val = $res->header('X-Foo');
239
240Shortcut for C<< $res->headers->header >>.
241
242=item content_type, content_length, content_encoding
243
244 $res->content_type('text/plain');
245 $res->content_length(123);
246 $res->content_encoding('gzip');
247
248Shortcut for the equivalent get/set methods in C<< $res->headers >>.
249
250=item redirect
251
252 $res->redirect($url);
253 $res->redirect($url, 301);
254
255Sets redirect URL with an optional status code, which defaults to 302.
256
257Note that this method doesn't normalize the given URI string. Users of
258this module have to be responsible about properly encoding URI paths
259and parameters.
260
261=item location
262
263Gets and sets C<Location> header.
264
265Note that this method doesn't normalize the given URI string in the
266setter. See above in C<redirect> for details.
267
268=item cookies
269
270 $res->cookies->{foo} = 123;
271 $res->cookies->{foo} = { value => '123' };
272
273Returns a hash reference containing cookies to be set in the
274response. The keys of the hash are the cookies' names, and their
275corresponding values are a plain string (for C<value> with everything
276else defaults) or a hash reference that can contain keys such as
277C<value>, C<domain>, C<expires>, C<path>, C<httponly>, C<secure>.
278
279C<expires> can take a string or an integer (as an epoch time) and
280B<does not> convert string formats such as C<+3M>.
281
282 $res->cookies->{foo} = {
283 value => 'test',
284 path => "/",
285 domain => '.example.com',
286 expires => time + 24 * 60 * 60,
287 };
288
289=item finalize
290
291 $res->finalize;
292
293Returns the status code, headers, and body of this response as a PSGI
294response array reference.
295
296=back
