40=head1 NAME
41
42Plack::Middleware::Static - serve static files with Plack
43
44=head1 SYNOPSIS
45
46 use Plack::Builder;
47
48 builder {
49 enable "Plack::Middleware::Static",
50 path => qr{^/(images|js|css)/}, root => './htdocs/';
51 $app;
52 };
53
54=head1 DESCRIPTION
55
56Enable this middleware to allow your Plack-based application to serve
57static files.
58
59If the given request matches with the pattern defined in C<path>, this
60middleware will try to locate the file in C<root>. If the file exists
61it will be served but otherwise C<404> response will be returned. See
62C<pass_through> option below to change this behavior.
63
64If the requested document is not within the C<root> (i.e. directory
65traversal) or the file is there but not readable, a 403 Forbidden
66response will be returned.
67
68The content type returned will be determined from the file extension
69based on L<Plack::MIME>.
70
71=head1 CONFIGURATIONS
72
73=over 4
74
75=item path, root
76
77 enable "Plack::Middleware::Static",
78 path => qr{^/static/}, root => 'htdocs/';
79
80C<path> specifies the URL pattern (regular expression) or a callback
81to match with requests to serve static files for. C<root> specifies
82the root directory to serve those static files from. The default value
83of C<root> is the current directory.
84
85This examples configuration serves C</static/foo.jpg> from
86C<htdocs/static/foo.jpg>. Note that the matched C</static/> portion is
87still appears in the local mapped path. If you don't like it, use a
88callback instead to munge C<$_>:
89
90 enable "Plack::Middleware::Static",
91 path => sub { s!^/static/!! }, root => 'static-files/';
92
93This configuration would serve C</static/foo.png> from
94C<static-files/foo.png> (not C<static-files/static/foo.png>). The
95callback specified in C<path> option matches against C<$_> and then
96updates the value since it does s///, and returns the number of
97matches, so it will pass through when C</static/> doesn't match.
98
99If you want to map multiple static directories from different root,
100simply add "this", middleware multiple times with different
101configuration options.
102
103=item pass_through
104
105By turning on this option, this middleware will pass the request
106back to the application for further processing, if the incoming
107request path matches with the C<path> but the requested file is not
108found on the file system.
109
110=back
